# Product

## Register

product

## Users

Dispatchers and mission controllers monitoring live security operations. Their primary context is real-time situational awareness: tracking guard positions, active missions, checkpoint progress, and incoming alerts. They work under time pressure and need to make decisions quickly based on accurate, up-to-the-second information.

## Product Purpose

GuardTour is a security guard tour management system that enables organizations to dispatch, monitor, and manage security personnel performing patrols. The dashboard (Operations Hub) serves as the mission control center — giving dispatchers a single-pane-of-glass view of who is on duty, what missions are active, where guards are, and what needs attention.

## Brand Personality

Professional, clean, reliable. The dashboard should feel like a trusted operations console — enterprise-grade, polished, and confidence-inspiring. Not flashy, not playful. Every element should communicate competence and control.

## Anti-references

- No gamified UI elements (badges, progress animations that feel trivial, celebratory graphics). This is serious security work, not a game.
- No overly decorative or ornamental design. Function over flourish.
- Avoid cluttered dashboards that obscure critical information.

## Design Principles

1. **Mission-first hierarchy** — The most critical operational data (active missions, guard status, alerts) is always the most prominent. Secondary information is accessible but visually subordinate.
2. **Clarity under pressure** — Dispatchers operate in time-sensitive environments. Labels, data, and actions must be immediately understandable without cognitive friction.
3. **Trust through polish** — Every pixel should reinforce reliability. Consistent spacing, crisp typography, intentional color use. Sloppy UI erodes trust in the system.
4. **Progressive disclosure** — Surface the essentials by default; let users drill into detail when needed. Never overwhelm the primary view.
5. **Dark-native, not dark-default** — The dark theme serves a real operational need (prolonged screen use in low-light control rooms). Every contrast decision should be intentional, not aesthetic.

## Accessibility & Inclusion

- WCAG AA as the baseline standard (4.5:1 text contrast minimum, 3:1 for large text).
- Color is never the sole indicator of status — use icons, labels, or patterns alongside color-coded status indicators.
- Support for reduced motion preferences.
- Keyboard-navigable controls for all interactive elements.
